Methodology: Each mode seeds the same collection structure (documents with username, email, score, status, tags, timestamps, metadata) then runs identical benchmarks. ID generation cost is included in write measurements — this is intentional, as auto-increment modes pay a real DB round-trip per ID via findOneAndUpdate. Bulk operations use batch ID allocation (one DB call per batch). Mode execution order is randomized to eliminate first-mover cache bias. Write concern is default (w:1).
  • Individual Write: Single-document inserts and upserts — shows per-operation overhead including ID generation
  • Bulk Write: insertMany in batches of 5,000 — ID generation excluded from timing, measures pure insert throughput
  • Read/Query: Point lookups, range scans, pagination, aggregation — measures index efficiency by ID type
  • Sustained: 30-second continuous write + mixed read/write with 10 concurrent workers
Reading the tables: Percentages show difference vs ObjectId (baseline). For ops/sec, green = faster. For latency, green = lower (faster), red = higher (slower).
